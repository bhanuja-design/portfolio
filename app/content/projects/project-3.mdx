# Project Three

Building a responsive analytics dashboard that works equally well on desktop and mobile.

## Problem

The legacy dashboard was designed for large screens only. Mobile users had to pinch, zoom, and scroll horizontally to access critical metrics, making quick checks impractical.

## Approach

I started with mobile wireframes, prioritizing the three metrics users checked most often. A flexible grid system and collapsible detail panels allowed the layout to scale up gracefully for tablet and desktop viewports.

## Outcome

Mobile usage of the dashboard grew significantly after launch. Stakeholders could monitor key numbers on the go, and the component library built during this project accelerated later dashboard work.
